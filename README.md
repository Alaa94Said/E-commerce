:file_folder: Repository Structure

This repository includes the following main components:

cypress/e2e: Contains test scenarios for complete purchase and checkout flows.

cypress/support/pages: Includes Page Object files for modular and reusable UI interactions.

cypress.config.js: Main configuration file for Cypress setup and Allure reporting integration.

screenshots/: Stores screenshots for failed test runs (auto-generated by Cypress).

allure-results/: Contains raw Allure report data after test execution.

:boom: What is it all about?

The goal of this project is to demonstrate a fully automated E2E test suite for an e-commerce platform using Cypress.
It simulates real-world user interactions such as account creation, login, adding products to the cart, submitting reviews, and completing the checkout process â€” all validated through assertions and integrated with Allure Reports for detailed visual analytics.

:golf: Why was this project created?

E-commerce systems involve complex user journeys that require reliable validation. This framework was created to:

Automate regression scenarios in the shopping flow.

Ensure consistent and accurate test execution through Cypress automation.

Provide clear, detailed reporting using Allure.

Support easy CI/CD integration with GitHub Actions or Azure DevOps pipelines.
